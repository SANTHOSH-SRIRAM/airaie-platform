# Backlog

Prioritized triage lists of deferred work. Each backlog has a fixed scope (a phase, a milestone, a hardening pass) and tracks items with severity + fix sketch — not a full plan.

| File | Covers |
|---|---|
| [`PHASE_G_HARDENING_BACKLOG.md`](./PHASE_G_HARDENING_BACKLOG.md) | 30 items deferred from Phases A–F, security/reliability/schema/frontend hardening. Sprints 1+2 closed 9/30. |

## When to add a backlog file

When a phase or sprint produces deferred items that don't fit a single phase plan. Backlog files are **categorized triage**, not execution plans — they're consulted when scoping the next sprint, and items are picked off into [`../phases/`](../phases/) or [`../quick/`](../quick/) entries when actively worked.

## When to retire a backlog file

When all items are either closed or migrated into another backlog/phase. Move to [`../archive/`](../archive/) with a closing summary.
