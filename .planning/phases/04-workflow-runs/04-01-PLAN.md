# Plan 04-01: Run Types, API, Hooks, Execution List, and Live DAG

<objective>
Define all run-related TypeScript types, build the API layer with realistic engineering mock data, create React Query hooks with polling for live updates, build a Zustand store for run UI state, implement the ExecutionList sidebar with status filters, build the RunDAGViewer that reuses WorkflowCanvas node types with status overlays, and create the WorkflowRunsPage that composes these into the full runs view. This wave delivers the core data layer and the two primary visual components (run list + live DAG) so Wave 2 can add log viewer, node detail, and action bar on top.
</objective>

<must_haves>
- Run types cover all states: running, succeeded, failed, waiting, cancelled
- API module provides list, detail, logs, cancel, retry with mock data fallback
- React Query hooks poll run detail every 3 seconds while status is running
- ExecutionList renders in left sidebar with All/Running/Succeeded/Failed filter tabs
- RunDAGViewer renders the same DAG as editor but with live node status overlays (green checkmark, blue spinner, gray pending, red failed)
- WorkflowRunsPage is routable and composes the full layout
- Run store tracks selectedRunId, selectedRunNodeId, and active status filter
</must_haves>

```ts
  nodeId: string;
  nodeName: string;
  nodeType: string;
  status: RunNodeStatus;
  startedAt?: string;
  completedAt?: string;
  duration?: number; // seconds
  inputs: Record<string, unknown>;
  outputs: Record<string, unknown> | null;
  metrics: RunNodeMetrics | null;
  attempts: RunNodeAttempt[];
}

export interface RunNodeMetrics {
  cpuPercent: number;
  memoryMb: number;

export interface RunNodeAttempt {
  attempt: number;
  status: RunNodeStatus;
  startedAt: string;
  completedAt?: string;
  duration?: number;
  error?: string;
}

export interface RunLogLine {
  timestamp: string;
  nodeId: string;
  nodeName: string;
  level: 'info' | 'warn' | 'error' | 'debug';
  message: string;
}

export interface RunDetail {
  id: string;
  workflowId: string;
  workflowName: string;
  status: RunStatus;
  startedAt: string;
  completedAt?: string;
  duration: number;
  costUsd: number;
```

```ts
import { create } from 'zustand';
import type { RunStatusFilter } from '@/types/run';

export interface RunStore {
  selectedRunId: string | null;
  selectedRunNodeId: string | null;
  statusFilter: RunStatusFilter;
  logAutoScroll: boolean;
  activeLogTab: 'logs' | 'artifacts' | 'cost' | 'timeline';

  selectRun: (runId: string | null) => void;
  selectRunNode: (nodeId: string | null) => void;
  setStatusFilter: (filter: RunStatusFilter) => void;
  setLogAutoScroll: (enabled: boolean) => void;
  setActiveLogTab: (tab: 'logs' | 'artifacts' | 'cost' | 'timeline') => void;
  reset: () => void;
}

export const useRunStore = create<RunStore>((set) => ({
  selectedRunId: null,
  selectedRunNodeId: null,
  statusFilter: 'all',
  logAutoScroll: true,
  activeLogTab: 'logs',

  selectRun: (runId) => set({ selectedRunId: runId, selectedRunNodeId: null }),
  selectRunNode: (nodeId) => set({ selectedRunNodeId: nodeId }),
  setStatusFilter: (filter) => set({ statusFilter: filter }),
  setLogAutoScroll: (enabled) => set({ logAutoScroll: enabled }),
  setActiveLogTab: (tab) => set({ activeLogTab: tab }),
  reset: () =>
    set({
      selectedRunId: null,
      selectedRunNodeId: null,
      statusFilter: 'all',
      logAutoScroll: true,
      activeLogTab: 'logs',
    }),
}));
```

<action>
Create `frontend/src/hooks/useRuns.ts`:

1. Define `runKeys` query key factory:
   - `all: ['runs']`
   - `list: (workflowId: string) => ['runs', 'list', workflowId]`
   - `detail: (runId: string) => ['runs', 'detail', runId]`
   - `logs: (runId: string) => ['runs', 'logs', runId]`

2. `useRunList(workflowId: string, statusFilter?: RunStatusFilter)`:
   - queryKey: `runKeys.list(workflowId)`
   - queryFn: `fetchRunList({ workflowId, status: statusFilter === 'all' ? undefined : statusFilter })`
   - staleTime: 10_000
   - refetchInterval: 15_000

3. `useRunDetail(runId: string | null)`:
   - queryKey: `runKeys.detail(runId!)`
   - queryFn: `fetchRunDetail(runId!)`
   - enabled: `!!runId`
   - refetchInterval: dynamically 3_000 when `data?.status === 'running'`, otherwise `false`

4. `useRunLogs(runId: string | null)`:
   - queryKey: `runKeys.logs(runId!)`
   - queryFn: `fetchRunLogs(runId!)`
   - enabled: `!!runId`
   - refetchInterval: dynamically 2_000 when run is active, otherwise `false`
   - Accept a second parameter `isRunning: boolean` to control the polling interval

5. `useCancelRun()`:
   - useMutation wrapping `cancelRun`
   - onSuccess: invalidate `runKeys.all`

6. `useRetryRun()`:
   - useMutation wrapping `retryRun`
   - onSuccess: invalidate `runKeys.all`
</action>
