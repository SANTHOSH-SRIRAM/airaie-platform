# Plan 06-02: Properties Panel, Action Bar, Top Bar, and Integration

<objective>
Build the ToolPropertiesPanel (version history, contract viewer, execution config, resource limits, sandbox policy), the ToolRegistryActionBar (Use in Workflow, View Contract, Test Run), and the ToolRegistryTopBar (title with badge, search, Register Tool button). Then wire all tool registry components into the existing AppShell slots: properties panel into RightPanel, action bar into BottomBar, filter sidebar into SidebarContentRouter, and update App.tsx routing to use ToolRegistryPage instead of the old ToolsPage.
</objective>

<must_haves>
- ToolPropertiesPanel shows selected tool's version history, contract (inputs/outputs with types), execution config, limits, and sandbox policy
- ToolRegistryActionBar renders "Use in Workflow", "View Contract", "Test Run" buttons plus tool count
- ToolRegistryTopBar renders title, tool count badge, search input, and "+ Register Tool" button
- RightPanel routes to ToolPropertiesPanel when contentType is 'tool-properties'
- BottomBar routes to ToolRegistryActionBar when contentType is 'tool-registry'
- SidebarContentRouter renders FilterSidebar in the 'filters' slot
- App.tsx /tools route renders ToolRegistryPage
- Selecting a tool opens the right panel with tool-properties content
</must_haves>

Use `data-testid="tool-properties-panel"` on root. Each section uses `data-testid="tool-properties-{section}"` (e.g., `tool-properties-contract`, `tool-properties-versions`).
