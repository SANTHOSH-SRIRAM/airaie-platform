# Plan 07-02: Performance, Responsive Layout, Accessibility, and Dark Mode

<objective>
Verify and optimize bundle size to stay under 500KB gzipped. Add responsive layout behavior so the sidebar auto-collapses at narrow viewports. Add comprehensive ARIA attributes, keyboard navigation, and a skip-to-content link for accessibility compliance. Verify and fix dark mode rendering across all new components by replacing hardcoded `bg-white` with CSS variable-backed Tailwind classes. Produce a final integration checklist.
</objective>

<must_haves>
- Vite build produces main chunk < 500KB gzipped with ReactFlow and recharts in separate chunks
- Sidebar auto-collapses when viewport width < 1280px
- Right panel is resizable (already done) and respects min/max constraints
- Skip-to-content link in AppShell that focuses the main content area
- ARIA labels on all interactive layout elements (sidebar, right panel, bottom bar, vertical toolbar)
- Keyboard navigation: Escape closes right panel, Tab cycles through sidebar items
- All hardcoded `bg-white` replaced with `bg-surface` or `bg-card-bg` from CSS variable tokens
- Dark mode renders correctly: no white backgrounds, no invisible text, proper borders
</must_haves>

<title>Add responsive sidebar auto-collapse at narrow viewports</title>
<read_first>
- frontend/src/components/layout/AppShell.tsx
- frontend/src/components/layout/Sidebar.tsx
- frontend/src/store/uiStore.ts
- frontend/src/styles/tokens.css (--sidebar-width)
</read_first>
<action>
1. In `frontend/src/components/layout/AppShell.tsx`, add a `useEffect` that listens to window resize events and auto-collapses the sidebar when `window.innerWidth < 1280`:
   ```tsx
   const toggleSidebar = useUiStore((s) => s.toggleSidebar);
   const sidebarCollapsed = useUiStore((s) => s.sidebarCollapsed);

   useEffect(() => {
     const handleResize = () => {
       if (window.innerWidth < 1280 && !sidebarCollapsed) {
         toggleSidebar();
       }
     };
     window.addEventListener('resize', handleResize);
     // Check on mount
     handleResize();
     return () => window.removeEventListener('resize', handleResize);
   }, []); // Run once on mount; sidebar toggle is imperative
   ```
   Note: Use a ref to track the previous state to avoid toggling on every resize event. Only collapse if not already collapsed.

2. In `frontend/src/store/uiStore.ts`, add a `collapseSidebar` action that sets `sidebarCollapsed: true` (one-way, avoids toggle oscillation):
   ```ts
   collapseSidebar: () => set({ sidebarCollapsed: true }),
   ```

3. In AppShell, use `collapseSidebar()` instead of `toggleSidebar()` in the resize handler to prevent oscillation.
</action>

   ```tsx
     const handleKeyDown = (e: KeyboardEvent) => {
       if (e.key === 'Escape' && rightPanel.open) {
         closeRightPanel();
       }
     };

   ```
