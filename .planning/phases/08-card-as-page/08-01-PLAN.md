# Plan 08-01 — Foundation

<objective>
Stand up the per-card route `/cards/:cardId` with a working layout shell, a per-card top bar (breadcrumb + title editor + Mode/status badges + Save/Run/⋯ actions, mirroring the Sensor Manager reference's "Deploy" pattern), and an augmented sidebar that injects "Cards in this Board" navigation and a "This Card Status" live pill while on the Card route. Wire `BoardDetailPage` to navigate on Card click, with a `?legacy=1` fallback to the existing side-sheet for one release. No body sections (Hero, Tables, Sequence, Status) yet — those come in 08-02. The page renders an empty body slot during this wave.
</objective>

<must_haves>
- `/cards/:cardId` route registered in `App.tsx`, lazy-loaded, wrapped in `ErrorBoundary`
- `cardDetailPath(id)` helper exported from `constants/routes.ts`
- `CardDetailPage` reads `useCard(cardId)` + `useBoard(card.board_id)` and renders an empty layout shell with `<CardTopBar>` and a placeholder body
- `CardTopBar` shows: ← back to Board, breadcrumb, title (inline-editable via `useUpdateCard`), Mode badge, status badge, `[Save]` (when dirty), `[▶ Run Card]` primary action, `[⋯]` overflow
- Mode badge color matches Govern hue (orange `#f57c00`)
- Status badge maps current backend statuses (`draft | ready | queued | running | completed | failed | blocked | skipped`) to existing color tokens
- `BoardDetailPage` Cards-tab click navigates to `cardDetailPath(card.id)`; existing side-sheet behavior available via `?legacy=1`
- Sidebar (via `useUiStore`) injects two new blocks while on `/cards/:id`:
  - `<ThisBoardNav>` — Cards in current Board, navigable, current Card highlighted
  - `<ThisCardStatusPill>` — live status: status dot + Mode + counts + last activity
- Vitest covers: CardTopBar renders correct Mode badge; ThisBoardNav lists cards and highlights active; BoardDetailPage navigation flow
- `tsc --noEmit` and `tsc --noEmit -p tsconfig.app.json` both clean
</must_haves>
