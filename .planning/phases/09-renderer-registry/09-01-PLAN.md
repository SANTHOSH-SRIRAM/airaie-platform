# Plan 09-01 — Renderer Registry MVP

<objective>
Replace `CardStatusPanel`'s artifact-list-with-download-links Results section with a **Renderer Registry** that picks the right component per `(intent_type, artifact_kind)` and mounts it inline. Phase 2a ships 5 renderers covering 80% of today's tools' output formats: image, JSON metrics, CSV-as-table, CSV-as-chart, and a generic fallback. Each renderer is its own lazy-loaded chunk. A per-`intent_type` layout config decides which renderers render in which slots (e.g. `cfd_analysis` → contour + residuals chart + metrics card; `parameter_sweep` → Pareto chart + samples table; everything else → stack each artifact with `pickRenderer()`). No backend changes; `papaparse` is the only new dependency (~7KB). Defers heavy renderers (3D CAD, PDF, scientific viz, streaming, view-state persistence) to Phase 2b–2f.
</objective>

<must_haves>
- Registry data structure (`Renderer[]`) with `match(ctx)` predicate and `lazy()` component
- Lookup priority: `manifest.renderer_hint` → `(intent_type, kind)` exact → kind-only → fallback
- 5 renderers: `ImageRenderer`, `JsonMetricsRenderer`, `CsvTableRenderer`, `CsvChartRenderer`, `GenericArtifactRenderer`
- Per-`intent_type` `ResultsLayout` config — at least 3 entries: `cfd_analysis`, `fea_static`, `parameter_sweep`. Everything else falls through to stacked auto-pick.
- `<ResultsSection>` component composes the layout: 12-column grid, slots assigned to renderers, every renderer fetches its own bytes via the artifact's presigned download URL (browser ↔ MinIO direct, kernel does not proxy)
- `CardStatusPanel.tsx` updated to mount `<ResultsSection>` instead of the current bullet list of download links — only when `run.status === 'completed'`
- Each renderer is a lazy chunk; `papaparse` is dynamically imported inside CSV renderers so Cards without CSV outputs don't pay for it
- New API function `getArtifactDownloadUrl(artifactId)` calling `GET /v0/artifacts/{id}/download-url` (it already exists per the kernel reference; just wrap it on the frontend)
- Tests:
  - `registry.test.ts` — lookup priority, fallback, manifest hint override
  - `results-layouts.test.ts` — layout selection per intent_type
  - `ResultsSection.test.tsx` — renders correct slots for each intent_type, falls through to stacked auto-pick when no layout match
- Vitest: ≥ 15 net-new tests
- `tsc --noEmit` (default) and `tsc --noEmit -p tsconfig.app.json` (strict) both clean
- Bundle: each renderer chunk lazy; main bundle delta < 5 KB (registry is tiny; renderers don't ship in main)
- `papaparse` installed in `frontend/package.json` and committed
</must_haves>

```ts
import type { LazyExoticComponent } from 'react';
import type { ArtifactRef } from '@/types/artifact';
import type { IntentSpec } from '@/types/intent';

export type RendererCtx = {
  artifact_kind: string;        // 'csv' | 'json' | 'png' | 'frd' | 'vtu' | 'gltf' | …
  intent_type: string;          // 'cfd_analysis' | 'fea_static' | 'parameter_sweep' | …
  vertical?: string;            // 'engineering' | 'science' | …
};

export type RendererProps = {
  artifact: ArtifactRef;        // metadata: id, name, kind, hash, size
  downloadUrl: string;          // presigned MinIO URL
  intent?: IntentSpec;          // for context-aware rendering
};

export type RendererId =
  | 'image'
  | 'json-metrics'
  | 'csv-table'
  | 'csv-chart'
  | 'fallback';

export type Renderer = {
  id: RendererId;
  match: (ctx: RendererCtx) => boolean;
  component: LazyExoticComponent<React.ComponentType<RendererProps>>;
};
```

Create `frontend/src/renderers/registry.ts` exporting:

  2. exact `(intent_type, kind)` match
  3. kind-only match
  4. fallback (always-last entry, returns true)

```ts
import type { ArtifactRef } from '@/types/artifact';

export type ResultsSlot = {
  span: 1 | 2 | 3 | 4 | 6 | 8 | 12;
  height?: 'auto' | 'compact' | 'expanded';
  match: (a: ArtifactRef) => boolean;
  rendererId?: string;          // optional override; else pickRenderer fires
  fallbackText?: string;        // shown if no artifact matches
};

export type ResultsLayout = {
  slots: ResultsSlot[];
};

export const resultsLayouts: Record<string, ResultsLayout> = {
  cfd_analysis: {
    slots: [
      { span: 8, height: 'expanded', match: a => a.kind === 'vtu' },
      { span: 4, match: a => a.name === 'residuals.csv', rendererId: 'csv-chart' },
      { span: 4, match: a => a.name === 'metrics.json', rendererId: 'json-metrics' },
    ],
  },
  fea_static: {
    slots: [
      { span: 8, height: 'expanded', match: a => a.kind === 'frd' },
      { span: 4, match: a => a.name === 'metrics.json', rendererId: 'json-metrics' },
    ],
  },
  parameter_sweep: {
    slots: [
      { span: 12, height: 'expanded', match: a => a.name === 'pareto.csv', rendererId: 'csv-chart' },
      { span: 12, match: a => a.name === 'samples.csv', rendererId: 'csv-table' },
    ],
  },
};

export function getLayoutForIntent(intentType: string): ResultsLayout | null {
  return resultsLayouts[intentType] ?? null;
}
```
