# Plan 10-05 — Board Canvas

## Scope

Mirror the Phase 10 Card-canvas pattern at the Board level. Boards become Notion-like pages with their own typed-governance block kinds (rollups, grids, pools) on top of free text. Reuses `<AirAirEditor>` and the slash-menu / cardCanvasContext infrastructure already shipping.

**Split into 3 sub-phases for shippability:**

- **10-05a** — Backend scaffolding (mechanical mirror of Card body endpoint)
- **10-05b** — Frontend page framework + first NodeView (CardsGrid)
- **10-05c** — Remaining 4 NodeViews + slash menu + auto-migration

## Out of scope

- Mode-rule per-block locks → 10-06
- Drag-drop palette → 10-04b
- Removing the legacy `?legacy=1` Board fallback → 10-07

## Board-specific block kinds

| Block | attrs | Data source | Display |
|---|---|---|---|
| `cardsGridBlock` | `{ filter?: string }` | `useBoardCards(boardId)` | Grid of cards (title + intent_type + status pill) — clickable to /cards/:id |
| `cardsGraphBlock` | `{}` | `getCardGraph(boardId)` | xyflow DAG of card dependencies |
| `gatesRollupBlock` | `{}` | `useGateList(boardId)` | List of gates with status badges (passed/failed/pending counts at top) |
| `evidenceRollupBlock` | `{}` | `useBoardEvidence(boardId)` (TBD if hook exists) | Recent evidence rows aggregated across cards |
| `artifactPoolBlock` | `{}` | `useArtifactList(boardId)` | Grid/list of board artifacts |

Cardinality: each is 0..N (a Board could have multiple grids with different filters, multiple rollups, etc. — generous). Slash menu surfaces all 5 plus the layout kinds.

## Backend (10-05a)

Mirror the cards body endpoint, file-by-file:

| Card pattern | Board mirror |
|---|---|
| `031_card_body_blocks.sql` | `032_board_body_blocks.sql` (alters `boards`) |
| `model.Card.BodyBlocks/BodyBlocksVersion` | `model.Board.BodyBlocks/BodyBlocksVersion` |
| `store.ErrCardBodyVersionConflict` | `store.ErrBoardBodyVersionConflict` |
| `CardStore.UpdateCardBody` | `BoardStore.UpdateBoardBody` |
| `service.ErrCardBodyVersionConflict` | `service.ErrBoardBodyVersionConflict` |
| `CardService.UpdateCardBody` | `BoardService.UpdateBoardBody` |
| `Handler.updateCardBody` | `Handler.updateBoardBody` |
| `PATCH /v0/cards/{id}/body` | `PATCH /v0/boards/{id}/body` |

Acceptance:
- `go build ./...` clean
- Migration applies idempotently
- `curl PATCH /v0/boards/{id}/body` accepts `{ body_blocks, expected_version }` and returns `{ body_blocks_version }` or 409 with `current_version`

## Frontend (10-05b + 10-05c)

### 10-05b
- New `frontend/src/types/boardBlocks.ts` mirroring `cardBlocks.ts` — discriminated union for the 5 Board block kinds + a shared `CardBodyDoc`-like `BoardBodyDoc`
- New `frontend/src/editor/extensions/boardBlockNodes.ts` factory — same shape as `typedBlockNodes.ts` but for board kinds
- New `frontend/src/editor/boardCanvasContext.ts` — `{ boardId, board, cards, gates }` (analog of cardCanvasContext)
- New `frontend/src/editor/boardMigration.ts` — `generateDefaultBoardBody(board, cards, gates, evidence, artifacts)` — emits a default doc with `cardsGrid + gatesRollup + evidenceRollup + artifactPool` blocks
- New `getBoardArtifactsCount`, `useUpdateBoardBody` hooks (mirror of `useUpdateCardBody`)
- New `CardsGridBlockView` — first real NodeView; the others stay as `TypedBlockPlaceholder` until 10-05c
- New `BoardCanvasPage` — feature-flagged via `?canvas=1` on `/boards/:id`; mirror of `CardCanvasPage`
- `BoardDetailPage` adds `?canvas=1` branch (lazy import of `BoardCanvasPage`)

### 10-05c
- 4 remaining NodeViews: `CardsGraphBlockView`, `GatesRollupBlockView`, `EvidenceRollupBlockView`, `ArtifactPoolBlockView`
- Update `getSlashMenuItems` to surface the 5 Board-specific kinds when the editor context is a Board (or always — slash menu doesn't currently know its scope; TBD whether to make it context-aware or just show all 16 governance kinds plus the 5 Board-specific = 21 total)

## Risks

| Risk | Mitigation |
|---|---|
| `boards` table schema may differ from `cards` schema | Mirror the migration shape verbatim; if column naming differs, document |
| `BoardStore` may not be a separate file like `CardStore` | Inspect `airaie-kernel/internal/store/postgres_boards.go` first; if it doesn't exist, the methods may live in `postgres.go` |
| Slash menu `getSlashMenuItems` is currently Card-scoped (uses cardinality based on Card-block presence) | 10-05c needs to extend the helper signature with a `scope: 'card' \| 'board'` parameter, OR the helper accepts the doc and infers from block presence |
| Auto-migration helper for Board canvas may want to query LIST endpoints (cards/gates/artifacts) at page mount — n+1 fetches | Make BoardCanvasPage do the same single-page-fetch pattern as CardCanvasPage |
