# Phase 260425-f10 Plan 01: Polish 3 Agent Playground UX bugs — Summary

Fixed three cosmetic / first-load defects in the Agent Playground that the Phase 5 audit surfaced — reload no longer spawns a duplicate session, the assistant byline renders the real agent name, and the Inspector "Messages" counter actually counts messages instead of base64 byte length.

## File diffs (1-line each)

| File | Change |
|------|--------|
| `frontend/src/pages/AgentPlaygroundPage.tsx` | Pull `isLoading` from `useSessionList`, gate auto-create effect on `!sessionListIsLoading`, add to deps, pass `agentName={agentDisplayName}` to `ChatInterface`. |
| `frontend/src/components/agents/ChatInterface.tsx` | Add optional `agentName` prop (default `'Agent'`), forward to each `<ChatMessage agentName={agentName} />`. |
| `frontend/src/components/agents/ChatMessage.tsx` | Add optional `agentName` prop (default `'Agent'`), delete `const AGENT_NAME = 'FEA Optimizer'`, render `{agentName}` in the agent-role byline. |
| `frontend/src/components/agents/InspectorPanel.tsx` | Import `extractMessages` from `@api/agentPlayground`, replace `activeSession.history?.length ?? 0` with `extractMessages(activeSession).length`. |
