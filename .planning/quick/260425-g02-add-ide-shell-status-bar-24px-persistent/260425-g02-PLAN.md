**Step 1 — Create `frontend/src/components/layout/StatusBar.tsx`:**

Write a self-contained functional component, default-exported, with this shape:

```typescript
import { Wifi, WifiOff, Circle } from 'lucide-react';
import { useExecutionStore, type RunStatus } from '@store/executionStore';
import { useAgentPlaygroundStore } from '@store/agentPlaygroundStore';
import { useAgent, useAgentVersions } from '@hooks/useAgents';

const APP_VERSION = (import.meta.env.VITE_APP_VERSION as string | undefined) ?? '0.1';

// Map RunStatus -> { label, dotClass } for the execution pill.
// Use Tailwind colour utilities only (project uses Tailwind 3 + arbitrary values).
const RUN_STATUS_PRESENTATION: Record<RunStatus, { label: string; dotClass: string }> = {
  IDLE:      { label: 'Idle',      dotClass: 'bg-[#6e7781]' },  // grey
  PENDING:   { label: 'Pending',   dotClass: 'bg-[#d29922]' },  // amber
  RUNNING:   { label: 'Running',   dotClass: 'bg-[#3fb950] animate-pulse' },
  PAUSED:    { label: 'Paused',    dotClass: 'bg-[#d29922]' },
  SUCCEEDED: { label: 'Succeeded', dotClass: 'bg-[#3fb950]' },
  FAILED:    { label: 'Failed',    dotClass: 'bg-[#f85149]' },
  CANCELED:  { label: 'Canceled',  dotClass: 'bg-[#8b949e]' },
};

export default function StatusBar() {
  const activeAgentId = useAgentPlaygroundStore((s) => s.activeAgentId);
  const runStatus = useExecutionStore((s) => s.runStatus);
  const sseConnected = useExecutionStore((s) => s.sseConnected);

  // Both hooks are enabled-gated on truthy id — passing null is safe and skips the query.
  const { data: agentData } = useAgent(activeAgentId);
  const { data: versionsData } = useAgentVersions(activeAgentId);

  // Derive version label using the same rule as AgentPlaygroundPage:51-57.
  let versionLabel: string | null = null;
  if (activeAgentId && versionsData && versionsData.length > 0) {
    const publishedMax = versionsData
      .filter((v) => v.status === 'published')
      .reduce((m, v) => Math.max(m, v.version), 0);
    const overallMax = versionsData.reduce((m, v) => Math.max(m, v.version), 0);
    const versionNum = publishedMax || overallMax || 1;
    const versionStatus = versionsData.find((v) => v.version === versionNum)?.status ?? 'draft';
    versionLabel = `v${versionNum} ${versionStatus}`;
  }

  const showAgentSection = !!activeAgentId;
  const presentation = RUN_STATUS_PRESENTATION[runStatus] ?? RUN_STATUS_PRESENTATION.IDLE;

  return (
    <div
      data-testid="status-bar"
      role="status"
      aria-label="Application status"
      className="shrink-0 h-6 px-3 flex items-center justify-between gap-3 bg-[#1f1f1f] text-[#d4d4d4] text-[12px] leading-none border-t border-[#2d2d2d] font-sans"
    >
      {/* LEFT cluster: agent identity + execution + sse */}
      <div className="flex items-center gap-3 min-w-0">
        {showAgentSection && (
          <>
            <span className="truncate max-w-[240px]" title={agentData?.name ?? ''}>
              {agentData?.name ?? 'Agent'}
            </span>
            {versionLabel && (
              <span
                className="px-1.5 py-0.5 rounded bg-[#2d2d2d] text-[#9ca3af] text-[11px] leading-none"
                data-testid="status-bar-version-chip"
              >
                {versionLabel}
              </span>
            )}
            <span className="w-px h-3 bg-[#3d3d3d]" aria-hidden />
            <span className="flex items-center gap-1.5" data-testid="status-bar-run-pill">
              <Circle size={8} className={`rounded-full ${presentation.dotClass}`} fill="currentColor" strokeWidth={0} />
              <span>{presentation.label}</span>
            </span>
            <span className="w-px h-3 bg-[#3d3d3d]" aria-hidden />
            <span
              className="flex items-center gap-1"
              data-testid="status-bar-sse"
              title={sseConnected ? 'Live updates connected' : 'Live updates disconnected'}
            >
              {sseConnected ? (
                <Wifi size={12} className="text-[#3fb950]" />
              ) : (
                <WifiOff size={12} className="text-[#6e7781]" />
              )}
            </span>
          </>
        )}
      </div>

      {/* RIGHT cluster: build version */}
      <div className="flex items-center gap-2 shrink-0">
        <span className="text-[#9ca3af]">AirAIE v{APP_VERSION}</span>
      </div>
    </div>
  );
}
```

Implementation notes — DO follow these EXACTLY:

1. **Height.** `h-6` is exactly 24px (Tailwind 3 default scale: 1.5rem × 16 = 24). Do NOT use `py-*` — that grows the strip. Pin height with `h-6` and use `leading-none` on text so glyphs don't push the box.

2. **Tonal language.** Use VS Code-ish dark strip: `bg-[#1f1f1f] text-[#d4d4d4] border-t border-[#2d2d2d]`. The rest of the app chrome uses light cards on `#f5f5f0`, but a dark status strip is the design-doc convention (Section 18 lists `Status Bar: #0969da (loaded) / #1f2328 (idle)` — `#1f1f1f` is in that family). Do NOT try to match the `bg-[#fbfaf9]` pill style of `BottomBar.tsx` — that's a floating action pill, not a status footer.

3. **runStatus enum is UPPERCASE.** The prompt mentions lowercase values; the actual store enum is `'PENDING' | 'RUNNING' | 'SUCCEEDED' | 'FAILED' | 'CANCELED' | 'PAUSED' | 'IDLE'`. Use the uppercase form. The presentation map above already does this. Do NOT add `'awaiting'` — it doesn't exist in the type.

4. **Hooks are gated.** `useAgent(null)` and `useAgentVersions(null)` are safe — the hooks have `enabled: !!id` guards. Pass `activeAgentId` directly; the React Query infrastructure will skip the network call when null. This is why the component can mount on every route without thrashing the API.

5. **Version label format.** Per the prompt: `v{n} {status}` like `v1 published`. Use the same selection rule as `AgentPlaygroundPage.tsx:51-57` (highest published version, falling back to highest overall, falling back to 1) so the status bar agrees with the playground's top bar. Inline the rule — do NOT introduce a shared util for two call sites yet.

6. **No new dependencies.** Only `lucide-react` icons. The `Circle` icon is used as a coloured status dot; `fill="currentColor"` + the dot's `bg-*` class produces a filled coloured circle without an SVG `fill` mismatch (the `bg-*` paints the box, and the icon fills via currentColor). Alternatively, replace the `<Circle/>` element with a plain `<span className={`w-2 h-2 rounded-full ${presentation.dotClass}`} />` if currentColor handling proves fiddly — both are acceptable, plain span is simpler and preferred.

7. **Accessibility.** `role="status"` so screen-readers announce updates. The pill has a `title` for hover. The icon-only Wifi has a `title` so it's not a mystery glyph. No keyboard interactions in this iteration — the doc's "click for details" features are deferred.

8. **`VITE_APP_VERSION` fallback.** The current Vite config does NOT define `VITE_APP_VERSION`, and `package.json` has `"version": "0.0.0"`. So in the running dev server, `import.meta.env.VITE_APP_VERSION` is `undefined`. The fallback `?? '0.1'` keeps the displayed string sensible (`AirAIE v0.1`) until someone wires the env var or vite-define. Do NOT add a `define: { 'import.meta.env.VITE_APP_VERSION': ... }` block to `vite.config.ts` in this task — that's a separate decision (release tooling) and out of scope.
