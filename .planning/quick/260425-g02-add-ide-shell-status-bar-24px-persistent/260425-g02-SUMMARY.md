## What Renders

```

```

- **Agent name + version chip** — only visible when `useAgentPlaygroundStore.activeAgentId` is set (the playground page sets/clears this on mount/unmount). Version chip uses the highest-published-then-highest-overall rule mirrored from `AgentPlaygroundPage.tsx:51-57`.

- **SSE icon** — green `<Wifi />` when `sseConnected===true`, grey `<WifiOff />` otherwise.
- **Build version** — right-edge label `AirAIE v{VITE_APP_VERSION ?? '0.1'}`, always visible.
- **Hidden** when `hideChrome===true` (studio routes: `/agent-studio/*`, `/workflow-studio/*`, `/workflow-runs/*`, `/agent-playground/*`, and `/boards/*` when `studioFullscreen`). Fully sharing the existing chrome-hide rule — no new gate.

## Note on `VITE_APP_VERSION`

The current `frontend/vite.config.ts` does **not** define `VITE_APP_VERSION` and `frontend/package.json` carries `"version": "0.0.0"`. Therefore `import.meta.env.VITE_APP_VERSION` is `undefined` at runtime and the hard-coded fallback `'0.1'` is what currently renders. Wire vite-define / read from `package.json` when release tooling lands — out of scope for this task.
