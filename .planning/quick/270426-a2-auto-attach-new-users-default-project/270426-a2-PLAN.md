<objective>
The Phase 10 canvas smoke test paused earlier because new users registered through `/v0/auth/register` arrive with **zero project memberships**. Every subsequent request — `GET /v0/cards/{id}`, `GET /v0/boards`, `GET /v0/tools` — gates on project membership and returns 403 PROJECT_ACCESS_DENIED. The current workaround (manually inserting a `project_members` row via SQL post-register) is fine for one-off testing but blocks any auto-provisioned dev / CI / staging environment.

Fix: in the `RegisterUser` service path, after the user row is committed, attach the user to a default project. The default project comes from `AIRAIE_AUTOATTACH_PROJECT_ID` env var; if unset, fall back to the first project in the DB (`SELECT id FROM projects ORDER BY created_at LIMIT 1`); if no projects exist at all, succeed registration with no membership (preserving the current 403 behavior — better than failing registration).

This is purely backend Go work — no frontend changes, no migrations. Existing users are unaffected.
</objective>

<must_haves>
- `RegisterUser` service path attaches the new user to a default project via the same DB transaction as the user insert
- Default project resolution: env var `AIRAIE_AUTOATTACH_PROJECT_ID` first, then `SELECT id FROM projects ORDER BY created_at LIMIT 1`
- New `Store.AttachUserToProject(ctx, tx, userID, projectID, role)` interface method + Postgres impl with `ON CONFLICT (user_id, project_id) DO NOTHING`
- Default role: `viewer` (read-only access; user can be promoted manually)
- If no projects exist, registration still succeeds; the user simply has no membership
- One test in `auth_test.go` covers the auto-attach path
- `cd airaie-kernel && go build ./...` clean
- `cd airaie-kernel && go test -short ./internal/...` passes (modulo the 4 pre-existing agent test failures documented in 10-01-SUMMARY)
- After fix: register a fresh user via curl, then `GET /v0/boards` with the new auth — returns 200, not 403
</must_haves>

```go
func (s *PostgresStore) AttachUserToProject(ctx context.Context, tx Tx, userID, projectID, role string) error {
    _, err := tx.ExecContext(ctx,
        `INSERT INTO project_members (user_id, project_id, role, created_at)
         VALUES ($1, $2, $3, NOW())
         ON CONFLICT (user_id, project_id) DO NOTHING`,
        userID, projectID, role)
    return err
}

func (s *PostgresStore) GetDefaultProjectID(ctx context.Context) (string, error) {
    var id string
    err := s.db.QueryRowContext(ctx,
        `SELECT id FROM projects ORDER BY created_at LIMIT 1`).Scan(&id)
    if errors.Is(err, sql.ErrNoRows) {
        return "", nil
    }
    return id, err
}
```
