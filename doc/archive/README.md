# Archive

Superseded or historical documents. Kept for traceability — git blame is not a substitute for prose context — but **not authoritative** for any current decision.

| Subdir | Contains |
|---|---|
| [`legacy/`](./legacy/) | Pre-2026-04-26 conceptual docs (replaced by [`../concepts/`](../concepts/)), old sprint plans, ATP-SPEC v0.1, the original MVP/n8n flow docs |
| [`old_doc/`](./old_doc/) | Pre-2025-Q4 implementation drafts: original frontend plan, centralized UI design system v0, Sprint 1 setup notes |

## When to read these

- Tracing why a decision was made: read the dated doc, then check git log for what changed.
- Onboarding: **don't.** Start at [`../concepts/`](../concepts/).
- Comparing approaches: only if the alternative actually shipped — otherwise it's noise.

## When to delete

Don't, unless reviewed by someone with project tenure. The cost of keeping stale docs is small; the cost of losing context for a future audit is large.
